Отчет по учебной практике: Разработка системы тарификации автомобильных перевозок

Введение в проектную деятельность

Практика была инициирована организационным собранием, в рамках которого осуществлялось формулирование целевых ориентиров и распределение проектной тематики. В качестве индивидуального проекта мной был выбран комплекс задач по разработке программного обеспечения для автоматизации процессов тарификации и планирования маршрутов в сфере автомобильных грузоперевозок.

Нормативно-правовая база проектирования

Второй этап практики был посвящен изучению регламентирующей документации, определяющей правовые и технические аспекты разработки программного обеспечения. Для работы с нормативной базой использовалась информационно-правовая система «КонсультантПлюс», позволяющая осуществлять целенаправленный поиск по следующим ключевым запросам:

«правовое регулирование автомобильных перевозок»
«тарифы на грузоперевозки»
«договор перевозки грузов»
«лицензирование транспортной деятельности»
«ГОСТ транспортная документация»
Анализ гражданского законодательства

Детальному изучению подверглись положения Гражданского кодекса Российской Федерации (Глава 40 «Перевозка», Глава 41 «Транспортная экспедиция»), регулирующие:

порядок заключения договоров перевозки
ответственность перевозчика и грузовладельца
тарифную политику и расчет стоимости услуг
претензионную работу и разрешение споров
Нормативы технического проектирования

В соответствии с требованиями Единой системы программной документации (ЕСПД) проведен анализ следующих стандартов:

ГОСТ 19.101-77 – определяет типовой набор программной документации, необходимой для системы тарификации перевозок

ГОСТ 28195-89 – устанавливает критерии оценки качества программных решений для транспортной логистики

ГОСТ Р ИСО/МЭК 12119-2000 – регламентирует требования к функциональности и надежности программных продуктов
